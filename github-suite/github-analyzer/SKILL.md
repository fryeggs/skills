# /github-analyzer - GitHub 16维度深度分析器 v2.0.0

基于 D01-D16 评分体系的开源项目深度分析器。输出量化评分（0-100）、四维价值评估、能力矩阵和 S/A/B/C/D 等级。

> 详细维度定义: `references/dimension-definitions.md`
> 评分体系与权重: `references/scoring-rubric.md`

## Instructions

### 5 种分析模式

**1. 深度调研 (Deep Research)** — 默认模式
- 16 维度全量分析（D01-D16），3 遍阅读：结构→逻辑→细节
- 输出: 16 维度报告 + 综合评分 + 四维价值 + 能力矩阵
- 适用: 技术选型、架构学习、投资评估

**2. 快速概览 (Quick Overview)**
- 关键文件扫描，重点 README/入口/核心模块
- 输出: 项目概要 + 技术栈 + 初步评级（6 核心维度抽样）
- 适用: 初步筛选、快速了解

**3. 对比分析 (Comparative)** — 入口模式，链式调用 github-comparator
- 多项目 16 维度横向对比
- 输出: 对比矩阵 + 雷达图数据 + 推荐结论
- 实际执行: 调用 `/github-comparator` 独立 SKILL

**4. 价值评估 (Value Assessment)**
- 四维价值深度评估: 技术/产品/生态/AI
- 输出: 四维价值报告 + 行业基准对比 + 风险标注
- 实际执行: 调用 `/github-valuator` 独立 SKILL

**5. 能力矩阵 (Capability Matrix)**
- 从 16 维度聚合为 6-10 能力域，★1-5 星制
- 输出: 能力矩阵表 + 每域详解 + 竞品定位图
- 基于深度调研结果生成，可独立请求

### 16 维度框架 (D01-D16)

**技术价值** (7维度):
D01 架构设计(8%) | D02 代码质量(7%) | D03 安全性(7%) | D04 API设计(7%)
D05 性能优化(6%) | D13 构建系统(5%) | D14 命名规范(4%)

**产品价值** (4维度):
D04 API设计(共享) | D08 跨平台(7%) | D10 通道架构(6%) | D11 文档体系(6%)

**生态价值** (4维度):
D06 依赖管理(6%) | D07 演进趋势(6%) | D12 测试质量(6%) | D15 生态系统(6%)

**AI/Agent 价值** (2维度):
D09 AI/Agent系统(7%) | D16 Skill/扩展(6%)

> 权重总和=100%（D04 在综合评分中只计一次 7%）
> 每维度含 5-8 子项，子项满分 10 分，详见 `references/scoring-rubric.md`

### 工作流程

```
Step 1: 需求确认
→ 确认项目 URL、分析模式
→ 深度调研时可选维度子集（默认全部 16 维度）
→ 对比/价值评估模式链式调用对应 SKILL

Step 2: 项目获取
→ 克隆或浏览项目代码
→ 识别语言、框架、Monorepo 结构

Step 3: 16 维度深潜
→ 按用户选择的维度子集逐一分析
→ 每维度: 指标采集 → 代码级证据 → 子项评分(×10) → 维度百分制
→ 参考 references/dimension-definitions.md 的分析步骤

Step 4: 评分汇总
→ 综合评分 = Σ(维度分 × 权重)
→ 四维价值 = 组内等权平均
→ 等级: S(≥90) A(≥80) B(≥70) C(≥60) D(<60)

Step 5: 能力矩阵生成
→ 16 维度聚合为 6-10 能力域
→ ★1-5 星成熟度评级
→ 关键量化指标标注

Step 6: 报告生成与存储
→ 汇总: 16 维度报告 + 综合评分卡 + 能力矩阵
→ 可选存入调研成果库 ~/Claude/📖 调研成果库/GitHub源码分析/
```

| 代号 | 维度 | 得分 | 等级 | 关键证据 |

| 代号 | 维度 | 得分 | 等级 | vs 行业基准 |

### 生态价值
| 代号 | 维度 | 得分 | 等级 | 风险等级 |
(D06/D07/D12/D15)

| 代号 | 维度 | 得分 | 等级 | 关键发现 |

### 约束

- 每个维度评分必须有代码级证据（文件路径/行数/统计数据）
- 不可凭印象打分，无证据的维度标注"证据不足"
- 快速概览模式仅评估 6 核心维度，其余标注"未评估"
- 对比/价值评估模式链式调用独立 SKILL，不重复实现
- 无 AI/Agent 代码的项目，D09/D16 标注"不适用"
